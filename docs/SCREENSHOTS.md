# 📸 Screenshot Guide for NodePress Docs

This guide tells you what screenshots to take.

And where to put them in the documentation.

---

## 🎯 Why Screenshots Matter

Screenshots help beginners feel safe.

When they see what to expect, they feel less scared.

Every screenshot should:

- Show ONE thing at a time
- Have a red circle or arrow around the important part
- Be clear and not blurry
- Show the actual NodePress interface

## 🛠️ Tools to Take Screenshots

| Your Computer  | Tool to Use              | How To                                      |
| -------------- | ------------------------ | ------------------------------------------- |
| 🪟 Windows     | Snipping Tool            | Press `Windows + Shift + S`                 |
| 🍎 Mac         | Screenshot Tool          | Press `Command + Shift + 4`                 |
| 🌐 Any browser | Full Page Screen Capture | Search for "full page screenshot extension" |

---

## 📐 Image Guidelines

| Setting       | Value                                 |
| ------------- | ------------------------------------- |
| File type     | PNG                                   |
| Max width     | 1200 pixels                           |
| Max file size | 500 KB                                |
| Naming        | All lowercase, use dashes             |
| Markdown tag  | `![Description](images/filename.png)` |

---

## ✅ Checklist

Before publishing any guide, check:

- [ ] All 17 screenshots are taken
- [ ] Files are in `docs/images/` folder
- [ ] File names match the guide references
- [ ] Red arrows/circles added to highlight key areas
- [ ] Screenshots are not blurry
- [ ] No private info visible (emails, passwords)
- [ ] Markdown alt text describes what's in the image

---

## 🔄 When to Update Screenshots

Take new screenshots when:

- The NodePress design changes
- The setup wizard changes
- New features are added
- A new version is released
