# Support

## Getting Help

There are several ways to get help with NodePress:

### Documentation

- [README](README.md) — Quick start and overview
- [CONTRIBUTING](CONTRIBUTING.md) — Development guide
- Project documentation in the `docs/` directory

### Community

- **GitHub Issues** — Report bugs and request features
- **Discussions** — Ask questions and share ideas
- **Stack Overflow** — Tag questions with `nodepress`

### Professional Support

For commercial support and consulting, contact support@nodepress.dev.

## Before Asking

1. Check the documentation for your question
2. Search existing GitHub issues (open and closed)
3. Search Stack Overflow for similar problems
4. Make sure you're running the latest version

## Filing a Bug

When filing a bug report, please include:

- NodePress version
- Node.js version
- Operating system
- Database type and version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs or error messages

## Feature Requests

We welcome feature requests! Please:

1. Check existing issues to avoid duplicates
2. Explain the problem you're trying to solve
3. Describe the proposed solution
4. Consider alternatives

## Security Issues

For security vulnerabilities, please see [SECURITY.md](SECURITY.md).
Do not report security issues publicly.
